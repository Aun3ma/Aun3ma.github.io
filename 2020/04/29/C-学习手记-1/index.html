C#语言作为一种功能强大的语言，在很多地方都有使用（.NET以及Unity等等），在这里总结一下最近的学习成果。

委托

使用委托时要先实例化，和类一样，使用new关键字产生委托的新实例，然后将一个或者多个与委托签名匹配的方法与委托实例关联。随后调用委托时，就会调用所有与委托实例关联的方法。

使用delegate关键字创建委托类型

public delegate void BoiledEventHandler(Object sender, BoiledEventArgs e);

创建委托实例并将方法绑定到委托上

BoiledEventHandler delegater = function_1;
delegater(sender, event_args);

当然，一个委托可以绑定多个方法，并将多个方法一次执行。

我们可以使用+=绑定方法，使用-=取消绑定。

BoiledEventHandler delegater = new BoiledEventHandler(function_1)
delegater += function_2
delegater -= function_1

事件

事件本身是委托类型，由于委托的优越性，使用事件可以让我们更好地处理侦听、观察这类行为，为事件处理提供便利。

使用event关键字声明一个事件

public event BoiledEventHandler Boiled;

事件的触发与参数

通常一个事件需要一个或多个触发函数，同时由于一个事件可以绑定多个方法，方法之间的参数各不相同，所以这里使用.NET库提供的一个带有泛型参数的时间处理委托。
public delegate void EventHandler(object sender,TEventArgs e); TEventArgs 是一个泛型参数

public class BoiledEventArgs : EventArgs
{
    public readonly int temperature;
    public BoiledEventArgs(int temperature)
    {
        this.temperature = temperature;
    }
}

protected virtual void OnBoiled(BoiledEventArgs e)
{
    if (Boiled != null)
    {
        Boiled(this, e);
    }
}

自动实现的属性

属性的声明

public class Product
{
    public decimal Price { get; set; }
}

属性的使用

Product p2 = new Product { Price = 3.0M };

扩展方法

扩展方法的声明

public static void Dump(this Product p)
{
    Console.WriteLine("Extension Method Dump: {0},{1}", p.Name, p.Price);
}

其中，参数类型Product即为可以使用该扩展方法的类型，注意添加this关键字。

扩展方法的使用

Product p = new Product();
p.Dump();

匿名类型

匿名类型就是指没有名字的类，这样的类不能被引用，且只能通过new关键字来创建。

匿名类型的优点

  1. 能够很方便的将一组数据封装到单个对象中，并且你不必去定义这个对象类型。
  2. 使用new关键字，创建简单方便。

匿名类型的使用

var py = new { Name = "Dainne", Price = 50.0M };

var隐式类型

我们看到刚才在声明py这个匿名类型实例时使用的是var关键字，那么var到底有什么用呢？

var隐式类型，我们也可以认为var就是不指定类型，之后编译器会帮助我们自动选择合适的数据类型，使用var可以有效提高编码效率。

var的使用规则

  1. 最终类型由编译器给出。
  2. 一般情况下，需要给与var类型的变量初始值。（不能为空值）
  3. 只能够声明一次。
  4. 不能作为参数或者返回值。
